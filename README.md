# Provenance Guard

A backend API for creative sharing platforms that classifies whether submitted text was written by a human or generated by AI. The system scores attribution confidence, shows transparency labels to readers, logs every decision, and gives creators a path to appeal misclassifications.

---

## Architecture

A creator submits text through `POST /submit`. The request passes through a rate limiter, then enters a two-signal detection pipeline. Both signals produce a score between 0.0 and 1.0 representing AI-likelihood. The scores are combined into a single confidence score, which drives a transparency label shown to readers. Every decision is written to a structured SQLite audit log. If a creator disputes the result, they submit a `POST /appeal` — this updates the content status to `under_review` and appends the appeal to the log.

```
POST /submit  (text + creator_id)
        │
        ▼
  Rate limiter  (Flask-Limiter)
        │
        ▼
  ┌─────────────────────────────────────────┐
  │           Detection pipeline            │
  │                                         │
  │   LLM classifier      Stylometric       │
  │   Groq — semantic     signal            │
  │   + style score       TTR, variance,    │
  │                       density score     │
  └──────────┬─────────────────┬────────────┘
             │   signal scores  │
             └────────┬─────────┘
                      ▼
             Confidence scorer
             0.5 × LLM + 0.5 × stylometric
                      │
                      ▼
             Transparency label builder
             high-confidence human / high-confidence AI / uncertain
                      │
                      ▼
             Audit log  (SQLite)
             decision + signals + label + timestamp
                      │
                      ▼
             API response
```

## Detection Signals

### Signal 1: LLM classifier (Groq)

Sends the submitted text to `llama-3.3-70b-versatile` with a system prompt asking for a single float between 0.0 and 1.0 representing AI-likelihood. The model assesses tone, phrasing, coherence, and semantic flow holistically.

**Why this signal:** An LLM can evaluate the overall feel of writing in a way that pure statistics cannot. It catches generic phrasing, overly smooth transitions, and the kind of structural predictability common in AI-generated text.

**Blind spot:** The model may misclassify polished human writing as AI-generated, and heavily edited AI text may fool it. It is also not proof of authorship — it can only assess how the text reads, not who wrote it.

**Implementation note:** The response is parsed with a `try/except`. If the model returns anything unparseable, the score falls back to `0.5` (uncertain) rather than crashing.

### Signal 2: Stylometric heuristics

A pure Python function that measures four structural features of the text, each normalized to 0.0–1.0 and averaged into a single score:

- **Sentence length variance** — AI text tends to have uniform sentence lengths. High variance maps toward 0.0 (human).
- **Type-token ratio (TTR)** — AI text tends toward lower vocabulary diversity. High TTR maps toward 0.0 (human).
- **Punctuation density** — Casual human writing uses more varied punctuation. Low density maps toward 1.0 (AI).
- **Coefficient of variation** — Measures sentence length uniformity. Low CV (uniform) maps toward 1.0 (AI).

A lexical marker adjustment is also applied: the presence of common AI phrasing patterns (`"furthermore"`, `"stakeholders"`, `"paradigm shift"`, etc.) raises the score, while casual human markers (`"honestly"`, `"probably"`, `"my friend"`, etc.) lower it.

**Why this signal:** It is structurally independent from the LLM — it measures writing patterns rather than meaning. This makes the combination more informative than either signal alone.

**Blind spot:** Human writers can write in a clean, uniform style (minimalist poetry, academic writing), and AI text can be edited to look more varied. Short texts also limit the reliability of variance-based features.

## Confidence Scoring

Both signals are combined with equal weighting:

```
combined_ai_score = (0.5 × llm_score) + (0.5 × stylometric_score)
```

Equal weighting was chosen because both signals measure genuinely different properties. Neither should dominate. If the two signals strongly disagree, the combined score falls into the uncertain range — which is the correct behavior, since mixed evidence should not produce a confident label.

### Thresholds

| Score range | Result |
|---|---|
| 0.00 – 0.25 | Likely human-written |
| 0.26 – 0.79 | Uncertain |
| 0.80 – 1.00 | Likely AI-generated |

The AI threshold (0.80) is intentionally higher than the human threshold (0.25). A false positive — labeling a human creator's work as AI-generated — is more harmful than a false negative on a creative platform.

### Example submissions from Milestone 4 testing

**High-confidence AI (score: 0.84)**
> "Artificial intelligence represents a transformative paradigm shift in modern society. It is important to note that while the benefits of AI are numerous, it is equally essential to consider the ethical implications. Furthermore, stakeholders across various sectors must collaborate to ensure responsible deployment."

LLM score: 0.80 | Stylometric score: 0.89 | Combined: **0.84** → Likely AI-generated

**High-confidence human (score: 0.05)**
> "ok so i finally tried that new ramen place downtown and honestly? underwhelming. the broth was fine but they put WAY too much sodium in it and i was thirsty for like three hours after."

LLM score: 0.10 | Stylometric score: 0.00 | Combined: **0.05** → Likely human-written

## Transparency Labels

The system displays one of three reader-facing labels based on the confidence score.

| Variant | Exact label text |
|---|---|
| High-confidence AI | "Likely AI-generated: Our system found strong indicators that this content may have been generated by AI. Because AI detection can be imperfect, the creator may appeal this classification." |
| High-confidence human | "Likely human-written: Our system found strong indicators that this content was written by a human creator." |
| Uncertain | "Uncertain: Our system could not confidently determine whether this content was written by a human or generated by AI." |

Labels are written in plain language for non-technical readers. The high-confidence AI label explicitly mentions the appeal option because a false positive on this label is the most harmful outcome.

## Rate Limiting

Rate limiting is applied to `POST /submit` using Flask-Limiter:

- **10 requests per minute**
- **50 requests per hour**

**Reasoning:** A real creator submits their own work infrequently — 10 per minute allows normal use while blocking automated flooding. The hourly limit prevents sustained abuse even from scripts that stay under the per-minute cap. These limits do not apply to `POST /appeal` or `GET /log`.

## Appeals Workflow

Any creator with a valid `content_id` can submit an appeal via `POST /appeal`.

**Request:**
```json
{
  "content_id": "e967d4fd-9644-4172-aca4-8a467839683a",
  "creator_reasoning": "I wrote this myself from personal experience. I am a non-native English speaker and my writing style may appear more formal than typical."
}
```

**Response:**
```json
{
  "appeal_id": "88552cd0-3f15-45d4-86b9-cfa61cdc6803",
  "content_id": "e967d4fd-9644-4172-aca4-8a467839683a",
  "status": "under_review",
  "message": "Appeal received. Your submission has been marked for review."
}
```

When an appeal is received the system: validates both fields, looks up the original decision, updates its status to `under_review`, and inserts a new appeal event into the audit log. Automated re-classification is not performed — the appeal is flagged for human review.

## API Reference

### POST /submit
Accept text content for attribution analysis.

**Request:**
```json
{"creator_id": "creator_123", "text": "Your text here."}
```

**Response:**
```json
{
  "content_id": "generated-uuid",
  "result": "Likely AI-generated",
  "confidence": 0.84,
  "ai_score": 0.84,
  "signals": [
    {"name": "llm_classifier", "score": 0.8},
    {"name": "stylometric_heuristics", "score": 0.89}
  ],
  "transparency_label": "Likely AI-generated: ..."
}
```

### POST /appeal
Contest a classification.

**Request:**
```json
{"content_id": "uuid", "creator_reasoning": "I wrote this myself."}
```

**Response:**
```json
{"appeal_id": "uuid", "content_id": "uuid", "status": "under_review", "message": "Appeal received."}
```

### GET /log
Returns the 50 most recent audit log entries as JSON.

## Known Limitations

**Short texts are unreliable.** The stylometric signal measures sentence length variance and type-token ratio — both of which are statistically meaningless with fewer than 3–4 sentences. A two-line poem will almost always land in the uncertain range regardless of its actual origin.

**Repetitive human writing scores high on stylometrics.** A human-written poem that intentionally uses repeated words and simple vocabulary will have a low type-token ratio and low sentence variance — both AI-like signals. The LLM signal may compensate, but the combined score could still push toward uncertain or even AI-generated.

**The lexical marker list is narrow.** The stylometric signal includes a fixed list of AI phrasing patterns. AI writing that avoids those specific phrases will not trigger the adjustment. This list needs ongoing maintenance to stay useful.

---

## Spec Reflection

**Where the spec helped:** The planning requirement to write out the three label variants before building anything was valuable. Having the exact label text decided upfront meant the `build_label()` function was trivial to implement — there were no design decisions left to make at implementation time.

**Where implementation diverged:** The spec described the stylometric signal as four pure statistical features. During testing, the normalized features alone produced mid-range scores for both human and AI text, so a lexical marker adjustment was added. This is still a stylometric heuristic — it measures repeated formal phrasing patterns — but it goes beyond what the original spec described. The change was necessary to produce meaningful score separation between clearly AI and clearly human text.

## AI Usage

**Instance 1: Flask app skeleton and LLM signal**
Provided the detection signals section and architecture diagram from `planning.md` and asked for a Flask app skeleton with a `POST /submit` route and an LLM signal function. The generated code used `temperature=1.0` on the Groq call, which was revised to `temperature=0.0` for deterministic scoring, and the fallback value on parse failure was changed from `None` to `0.5` to ensure uncertain behavior rather than a crash.

**Instance 2: Stylometric signal recalibration**
After the initial stylometric signal produced mid-range scores for all inputs, asked for a recalibration of the normalization caps. The generated caps were still too loose, so they were tightened further through manual testing. A lexical marker adjustment was then designed and added to improve separation on clearly AI-generated text — this was directed rather than generated.
